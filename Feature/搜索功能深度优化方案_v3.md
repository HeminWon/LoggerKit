# LoggerKit 搜索功能深度优化方案 v3

> **基于 v2 的深度审查和改进**
> 修复了架构违规、内存问题、用户体验问题

## 功能定位

**本方案将完全替换现有的 SearchFeature 功能：**

**现有功能（SearchFeature）：**
- 功能定位：搜索建议（Search Suggestions）
- 数据源：内存中最近 10000 条日志
- 工作方式：在内存中快速搜索 → 返回分类建议（message、fileName、function 等）
- 使用场景：帮助用户快速添加筛选条件

**新功能（本方案）：**
- 功能定位：深度日志搜索（Deep Log Search）
- 数据源：所有历史日志（基于数据库）
- 工作方式：渐进式搜索（Preview → 用户决定 → Full Search）→ 直接展示匹配的日志内容
- 使用场景：在海量日志中查找特定内容，查看完整日志详情

**替换理由：**
1. 数据库层搜索性能更优（5x 提升）
2. 支持海量日志搜索，不受内存限制
3. 提供更直观的搜索结果展示
4. 保留快速添加筛选器的能力（通过结果列表操作）

**迁移影响：**
- State 结构变化：`cachedSearchResults` → `previewResults` + `fullSearchResults`
- UI 变化：从分类建议列表 → 日志内容列表
- 用户操作：保持一致（输入关键词 → 查看结果 → 添加筛选器）

## 核心改进点

基于原方案的四个关键优化 + 深度改进：

1. ✅ **按 Session 渐进式搜索** - 从最新的 session 开始，逐个往前扫描（而非固定条数分页）
2. ✅ **输入防抖优化** - 快速输入时显示加载状态，避免频繁查询
3. ✅ **智能取消机制** - 更新关键词时立即取消上一次查询
4. ✅ **智能 Preview 范围** - 无论选中多少个 session，Preview 始终只搜索最新 3 个
5. 🆕 **用户主动触发 Full Search** - Preview 完成后显示"搜索更多"按钮，由用户决定是否继续
6. 🆕 **严格 TCA 架构** - 所有异步操作使用 Effect，无 DispatchQueue
7. 🆕 **搜索快照机制** - 搜索开始时创建快照，保证数据一致性
8. 🆕 **完善内存保护** - Preview 和 Full Search 都有结果数量限制

## 搜索流程设计

```
Idle → Typing → Preview (最新 3 个 session) → [用户点击"搜索更多"] → Full Search → Completed
        ↑              ↑                                ↑                         ↑
     300ms 防抖    立即执行                        用户主动触发              按 session 逐步搜索
```

## 1. 搜索状态机重新设计

```swift
public enum SearchPhase: Equatable, Sendable {
    /// 空闲 - 未搜索
    case idle

    /// 正在输入 - 等待用户停止输入（不执行查询）
    case typing

    /// 输入预览 - 搜索最新的 N 个 session
    /// - sessionCount: 预览的 session 数量
    case previewSearching(sessionCount: Int)

    /// 预览完成 - 等待用户决定是否搜索更多
    /// - matchCount: 预览找到的匹配数
    /// - searchedSessions: 已搜索的 session 数
    /// - hasMoreSessions: 是否还有更多 session 可搜索
    case previewCompleted(matchCount: Int, searchedSessions: Int, hasMoreSessions: Bool)

    /// 完整搜索中 - 按 session 逐步往前搜索
    /// - currentSessionIndex: 当前搜索到第几个 session（从0开始）
    /// - totalSessions: 总 session 数量
    /// - matchCount: 已找到的匹配数量
    /// - scannedEvents: 已扫描的日志事件数量
    case fullSearching(
        currentSessionIndex: Int,
        totalSessions: Int,
        matchCount: Int,
        scannedEvents: Int
    )

    /// 搜索完成
    /// - totalMatches: 总匹配数
    /// - searchedSessions: 搜索的 session 数量
    case completed(totalMatches: Int, searchedSessions: Int)

    /// 搜索取消
    case cancelled

    /// 搜索失败
    /// - message: 错误信息
    case failed(message: String)

    /// 结果过多，需要优化关键词
    /// - currentCount: 当前结果数
    /// - limit: 限制数量
    case tooManyResults(currentCount: Int, limit: Int)
}
```

```swift
/// 当前选中的 session IDs（从 FilterFeature 同步）
public var selectedSessionIds: Set<String> = []

/// 所有可用的 session IDs（从外部同步）
public var allAvailableSessionIds: Set<String> = []

// MARK: - Timing

/// Typing 防抖延迟（毫秒）
public let typingDebounceDelay: Int = 300

// MARK: - New Result Tracking (使用 Effect 管理)

/// 新结果的 IDs（用于 UI 高亮，通过 Effect 定时清除）
public var newResultIds: Set<String> = []

// MARK: - Computed Properties

/// 所有搜索结果（供 UI 展示，优化为存储属性）
public var allSearchResults: [LogEvent] {
    switch searchPhase {
    case .idle, .typing:
        return []

    case .previewSearching, .previewCompleted:
        return previewResults

    case .fullSearching, .completed:
        // 已经是有序的，直接合并
        return mergeOrderedResults(previewResults, fullSearchResults)

    case .failed, .tooManyResults, .cancelled:
        return []
    }
}

/// 合并两个有序数组（按时间倒序）
private func mergeOrderedResults(
    _ preview: [LogEvent],
    _ full: [LogEvent]
) -> [LogEvent] {
    var result: [LogEvent] = []
    var i = 0, j = 0

    while i < preview.count && j < full.count {
        if preview[i].timestamp > full[j].timestamp {
            result.append(preview[i])
            i += 1
        } else {
            result.append(full[j])
            j += 1
        }
    }

    result.append(contentsOf: preview[i...])
    result.append(contentsOf: full[j...])

    return result
}

/// 搜索进度百分比 (0.0 - 1.0)
public var searchProgress: Double {
    guard case .fullSearching(let currentIndex, let total, _, _) = searchPhase else {
        return 0.0
    }
    return total > 0 ? Double(currentIndex + 1) / Double(total) : 0.0
}

/// 是否正在搜索
public var isSearching: Bool {
    switch searchPhase {
    case .typing, .previewSearching, .fullSearching:
        return true
    default:
        return false
    }
}

/// 是否正在完整搜索
public var isFullSearching: Bool {
    if case .fullSearching = searchPhase {
        return true
    }
    return false
}

// MARK: - State Mutations (移到 Reducer)

/// 清空搜索结果
public mutating func clearSearchResults() {
    previewResults = []
    fullSearchResults = []
    searchSnapshot = nil
    newResultIds = []
}
```

## 5. Reducer 核心逻辑

### 5.1 Filter 状态同步

```swift
case .syncFilterState(let selectedIds, let allIds):
    state.selectedSessionIds = selectedIds
    state.allAvailableSessionIds = allIds

    // 如果正在搜索，且 filter 发生了重大变化，取消当前搜索
    if state.isSearching {
        let snapshotIds = state.searchSnapshot?.selectedSessionIds ?? []
        if snapshotIds != selectedIds {
            return .send(.cancelAllSearches)
        }
    }

    return .none
```

### 5.2 输入处理 - 防抖机制

```swift
case .updateSearchText(let text):
    state.searchText = text

    if text.isEmpty {
        // 清空搜索
        state.searchPhase = .idle
        state.clearSearchResults()
        return .merge(
            .cancel(id: CancellationId.typingDebounce),
            .cancel(id: CancellationId.previewSearch),
            .cancel(id: CancellationId.fullSearch),
            .cancel(id: CancellationId.newResultBadgeTimer)
        )
    }

    // 1. 清空旧的搜索结果（如果有）
    // ✅ 修复边界情况：用户修改关键词时，清空之前的搜索结果
    if state.searchPhase != .idle {
        state.clearSearchResults()
    }

    // 2. 立即进入 typing 状态
    state.searchPhase = .typing

    // 3. 取消之前的所有搜索
    let cancelEffects: Effect<Action> = .merge(
        .cancel(id: CancellationId.typingDebounce),
        .cancel(id: CancellationId.previewSearch),
        .cancel(id: CancellationId.fullSearch),
        .cancel(id: CancellationId.newResultBadgeTimer)
    )

    // 4. 启动 typing 防抖定时器（300ms）
    let typingDebounce = Effect<Action>.run { send in
        try await Task.sleep(nanoseconds: UInt64(state.typingDebounceDelay * 1_000_000))
        await send(.startPreviewSearch)
    }
    .cancellable(id: CancellationId.typingDebounce)

    return .merge(cancelEffects, typingDebounce)
```

### 5.3 Preview 搜索 - 固定搜索最新 3 个 session

```swift
case .startPreviewSearch:
    guard !state.searchText.isEmpty else { return .none }

    state.searchPhase = .previewSearching(sessionCount: state.previewSessionCount)

    return handlePreviewSearch(state: state, environment: environment)

private func handlePreviewSearch(
    state: State,
    environment: Environment
) -> Effect<Action> {
    let searchText = state.searchText
    let searchFields = state.searchFields
    let selectedSessionIds = state.selectedSessionIds
    let allAvailableSessionIds = state.allAvailableSessionIds
    let previewSessionCount = state.previewSessionCount
    let resultsLimit = state.searchResultsLimit

    return .run { [dataLoader = environment.dataLoader] send in
        do {
            // 1. 创建搜索快照（保证数据一致性）
            let snapshot = try await createSearchSnapshot(
                dataLoader: dataLoader,
                searchText: searchText,
                searchFields: searchFields,
                selectedSessionIds: selectedSessionIds,
                allAvailableSessionIds: allAvailableSessionIds,
                previewSessionCount: previewSessionCount
            )

            // 2. 查询 Preview sessions 的所有日志（使用数据库搜索，而非加载全部）
            let matches = try await dataLoader.searchEvents(
                sessionIds: Set(snapshot.previewSessions.map { $0.id }),
                searchText: searchText,
                searchFields: searchFields,
                limit: resultsLimit
            )

            // 3. 检查结果数量
            if matches.count > resultsLimit {
                await send(.searchResultsExceeded(currentCount: matches.count, stage: .preview))
                return
            }

            // 4. 完成 Preview
            await send(.previewSearchCompleted(
                matches: matches,
                searchedSessions: snapshot.previewSessions.count,
                hasMoreSessions: !snapshot.fullSearchSessions.isEmpty
            ))

        } catch {
            await send(.previewSearchFailed(error))
        }
    }
    .cancellable(id: CancellationId.previewSearch)
}

/// 创建搜索快照
private func createSearchSnapshot(
    dataLoader: DataLoader,
    searchText: String,
    searchFields: Set<SearchField>,
    selectedSessionIds: Set<String>,
    allAvailableSessionIds: Set<String>,
    previewSessionCount: Int
) async throws -> SearchSnapshot {

    // 确定搜索范围
    let searchSessionIds: Set<String>
    if selectedSessionIds.isEmpty || selectedSessionIds.count == allAvailableSessionIds.count {
        // 未选或全选 → 搜索所有
        searchSessionIds = allAvailableSessionIds
    } else {
        // 选中特定 session → 只搜索这些
        searchSessionIds = selectedSessionIds
    }

    // 获取所有 sessions（按时间倒序）
    let allSessions = try await dataLoader.getSessions(
        sessionIds: searchSessionIds,
        sortBy: .timeDescending
    )

    // 🔧 修复：无论用户选了多少个 session，Preview 始终只取最新 3 个
    let previewSessions = Array(allSessions.prefix(previewSessionCount))
    let fullSearchSessions = Array(allSessions.dropFirst(previewSessionCount))

    return SearchSnapshot(
        searchText: searchText,
        searchFields: searchFields,
        selectedSessionIds: selectedSessionIds,
        allAvailableSessionIds: allAvailableSessionIds,
        allSessions: allSessions,
        previewSessions: previewSessions,
        fullSearchSessions: fullSearchSessions,
        createdAt: Date()
    )
}

case .previewSearchCompleted(let matches, let sessions, let hasMore):
    state.previewResults = matches
    state.searchPhase = .previewCompleted(
        matchCount: matches.count,
        searchedSessions: sessions,
        hasMoreSessions: hasMore
    )
    return .none

case .previewSearchFailed(let error):
    state.searchPhase = .failed(message: error.localizedDescription)
    return .none
```

### 5.4 用户主动触发 Full Search

```swift
case .userRequestedFullSearch:
    guard let snapshot = state.searchSnapshot else {
        return .none
    }

    guard !snapshot.fullSearchSessions.isEmpty else {
        // 没有更多 session 需要搜索
        return .none
    }

    return handleFullSearch(state: state, snapshot: snapshot, environment: environment)

private func handleFullSearch(
    state: State,
    snapshot: SearchSnapshot,
    environment: Environment
) -> Effect<Action> {
    let resultsLimit = state.searchResultsLimit
    let alreadyFoundCount = state.previewResults.count

    return .run { [dataLoader = environment.dataLoader] send in
        do {
            let totalSessions = snapshot.fullSearchSessions.count
            var allMatches: [LogEvent] = []
            var scannedEvents = 0

            // 逐个 session 搜索
            for (index, session) in snapshot.fullSearchSessions.enumerated() {
                // 检查取消
                try Task.checkCancellation()

                // 更新进度
                await send(.updateFullSearchProgress(
                    currentSessionIndex: index,
                    totalSessions: totalSessions,
                    matchCount: allMatches.count,
                    scannedEvents: scannedEvents
                ))

                // 🔧 使用数据库搜索而非加载全部日志
                let sessionMatches = try await dataLoader.searchEvents(
                    sessionIds: [session.id],
                    searchText: snapshot.searchText,
                    searchFields: snapshot.searchFields,
                    limit: resultsLimit
                )

                scannedEvents += sessionMatches.count
                allMatches.append(contentsOf: sessionMatches)

                // 检查总结果数量（Preview + Full Search）
                if alreadyFoundCount + allMatches.count > resultsLimit {
                    await send(.searchResultsExceeded(
                        currentCount: alreadyFoundCount + allMatches.count,
                        stage: .fullSearch
                    ))
                    return
                }

                // 每个 session 完成后立即更新
                await send(.fullSearchBatchCompleted(
                    newMatches: sessionMatches,
                    currentSessionIndex: index,
                    totalSessions: totalSessions
                ))
            }

            // 全部完成
            await send(.fullSearchCompleted(
                totalMatches: allMatches.count,
                searchedSessions: totalSessions
            ))

        } catch is CancellationError {
            await send(.fullSearchCancelled)
        } catch {
            await send(.fullSearchFailed(error))
        }
    }
    .cancellable(id: CancellationId.fullSearch)
}

case .updateFullSearchProgress(let index, let total, let matches, let scanned):
    state.searchPhase = .fullSearching(
        currentSessionIndex: index,
        totalSessions: total,
        matchCount: matches,
        scannedEvents: scanned
    )
    return .none

case .fullSearchBatchCompleted(let newMatches, let index, let total):
    // 🔧 按时间倒序插入（维护有序性）
    state.fullSearchResults = insertOrderedMatches(
        existing: state.fullSearchResults,
        new: newMatches
    )

    // 标记新结果
    let newIds = Set(newMatches.map { $0.id })
    state.newResultIds.formUnion(newIds)

    // 更新进度
    let totalMatches = state.previewResults.count + state.fullSearchResults.count
    state.searchPhase = .fullSearching(
        currentSessionIndex: index,
        totalSessions: total,
        matchCount: totalMatches,
        scannedEvents: state.fullSearchResults.count
    )

    // 🔧 使用 Effect 定时清除新标记（而非 DispatchQueue）
    return .run { send in
        try await Task.sleep(nanoseconds: 3_000_000_000) // 3秒
        await send(.clearNewResultBadges(newIds))
    }
    .cancellable(id: CancellationId.newResultBadgeTimer, cancelInFlight: false)

/// 按时间倒序插入新结果
private func insertOrderedMatches(
    existing: [LogEvent],
    new: [LogEvent]
) -> [LogEvent] {
    let sorted = new.sorted { $0.timestamp > $1.timestamp }
    var result = existing

    for event in sorted {
        // 二分插入
        let index = result.firstIndex { $0.timestamp < event.timestamp } ?? result.count
        result.insert(event, at: index)
    }

    return result
}

case .clearNewResultBadges(let ids):
    state.newResultIds.subtract(ids)
    return .none

case .fullSearchCompleted(let matches, let sessions):
    let totalMatches = state.previewResults.count + matches
    state.searchPhase = .completed(
        totalMatches: totalMatches,
        searchedSessions: state.searchSnapshot?.previewSessions.count ?? 0 + sessions
    )
    return .none

case .fullSearchFailed(let error):
    state.searchPhase = .failed(message: error.localizedDescription)
    return .none

case .searchResultsExceeded(let count, _):
    state.searchPhase = .tooManyResults(currentCount: count, limit: state.searchResultsLimit)
    state.clearSearchResults()
    return .cancel(id: CancellationId.fullSearch)

case .cancelAllSearches:
    state.searchPhase = .cancelled
    return .merge(
        .cancel(id: CancellationId.typingDebounce),
        .cancel(id: CancellationId.previewSearch),
        .cancel(id: CancellationId.fullSearch),
        .cancel(id: CancellationId.newResultBadgeTimer)
    )

case .fullSearchCancelled:
    state.searchPhase = .cancelled
    return .none
```

### 🔴 P0 - 必须修复

#### P0-1: 内存双重占用问题

**问题描述：**

在 `handleFullSearch` 实现中（5.4 节 Line 495-555），存在内存双重占用：

```swift
// ❌ 问题代码
private func handleFullSearch(...) -> Effect<Action> {
    return .run { [dataLoader = environment.dataLoader] send in
        var allMatches: [LogEvent] = []  // 局部累积数组

        for (index, session) in snapshot.fullSearchSessions.enumerated() {
            let sessionMatches = try await dataLoader.searchEvents(...)

            allMatches.append(contentsOf: sessionMatches)  // ❌ 累积到局部

            await send(.fullSearchBatchCompleted(
                newMatches: sessionMatches,  // ❌ 同时发送到 state
                ...
            ))
        }
    }
}

// Reducer 处理
case .fullSearchBatchCompleted(let newMatches, ...):
    state.fullSearchResults = insertOrderedMatches(
        existing: state.fullSearchResults,
        new: newMatches  // ❌ 又插入到 state
    )
```

**风险分析：**
- 如果搜索 100 个 session，每个 1000 条日志
- `allMatches` 会累积到 10 万条记录
- `state.fullSearchResults` 也会包含相同的 10 万条
- **总内存占用 = 2x 数据量**，直到 Full Search 完成才释放 `allMatches`

**修复方案：**

```swift
// ✅ 修复后的代码
private func handleFullSearch(
    state: State,
    snapshot: SearchSnapshot,
    environment: Environment
) -> Effect<Action> {
    let resultsLimit = state.searchResultsLimit
    let alreadyFoundCount = state.previewResults.count

    return .run { [dataLoader = environment.dataLoader] send in
        do {
            let totalSessions = snapshot.fullSearchSessions.count
            // ✅ 移除 allMatches 局部变量

            for (index, session) in snapshot.fullSearchSessions.enumerated() {
                try Task.checkCancellation()

                let sessionMatches = try await dataLoader.searchEvents(
                    sessionIds: [session.id],
                    searchText: snapshot.searchText,
                    searchFields: snapshot.searchFields,
                    limit: resultsLimit
                )

                // ✅ 每个 session 完成后立即发送，让 Reducer 更新 state
                await send(.fullSearchBatchCompleted(
                    newMatches: sessionMatches,
                    currentSessionIndex: index,
                    totalSessions: totalSessions
                ))

                // ✅ 在 Reducer 中从 state 获取当前总数进行检查
                // 不再需要在 Effect 中累积 allMatches
            }

            // ✅ 完成时不需要传递总数，Reducer 从 state 计算
            await send(.fullSearchCompleted(searchedSessions: totalSessions))

        } catch is CancellationError {
            await send(.fullSearchCancelled)
        } catch {
            await send(.fullSearchFailed(error))
        }
    }
    .cancellable(id: CancellationId.fullSearch)
}
```

**Reducer 中相应修改：**

```swift
// 修改 Action 定义
case fullSearchCompleted(searchedSessions: Int)  // 移除 totalMatches 参数

// Reducer 处理
case .fullSearchBatchCompleted(let newMatches, let index, let total):
    // 插入新结果
    state.fullSearchResults = insertOrderedMatches(
        existing: state.fullSearchResults,
        new: newMatches
    )

    // 标记新结果
    let newIds = Set(newMatches.map { $0.id })
    state.newResultIds.formUnion(newIds)

    // ✅ 从 state 计算总匹配数
    let totalMatches = state.previewResults.count + state.fullSearchResults.count

    // ⚠️ 检查是否超过限制
    if totalMatches > state.searchResultsLimit {
        return .merge(
            .send(.searchResultsExceeded(currentCount: totalMatches, stage: .fullSearch)),
            .cancel(id: CancellationId.fullSearch)
        )
    }

    // 更新进度
    state.searchPhase = .fullSearching(
        currentSessionIndex: index,
        totalSessions: total,
        matchCount: totalMatches,
        scannedEvents: state.fullSearchResults.count
    )

    // 定时清除新标记
    return .run { send in
        try await Task.sleep(nanoseconds: 3_000_000_000)
        await send(.clearNewResultBadges(newIds))
    }
    .cancellable(id: CancellationId.newResultBadgeTimer, cancelInFlight: false)

case .fullSearchCompleted(let sessions):
    // ✅ 从 state 计算总匹配数
    let totalMatches = state.previewResults.count + state.fullSearchResults.count
    state.searchPhase = .completed(
        totalMatches: totalMatches,
        searchedSessions: state.searchSnapshot?.previewSessions.count ?? 0 + sessions
    )
    return .none
```

**优势：**
- ✅ 消除内存双重占用
- ✅ 内存使用降低 50%
- ✅ 逻辑更清晰，单一数据源（state）

#### P0-2: 快照生命周期管理不明确

**问题描述：**

在 `.cancelled`、`.failed`、`.tooManyResults` 等最终状态时，`searchSnapshot` 没有被明确清理。如果用户在这些状态下修改搜索词，快照可能污染新搜索。

**当前代码：**

```swift
// ❌ 快照未清理
case .cancelAllSearches:
    state.searchPhase = .cancelled
    return .merge(...)  // 没有清理 searchSnapshot

case .searchResultsExceeded(let count, _):
    state.searchPhase = .tooManyResults(...)
    state.clearSearchResults()  // ✅ 这里会清理
    return .cancel(...)

case .previewSearchFailed(let error):
    state.searchPhase = .failed(...)
    return .none  // ❌ 没有清理 searchSnapshot
```

**修复方案：**

```swift
// ✅ 在所有最终状态显式清理快照
case .cancelAllSearches:
    state.searchPhase = .cancelled
    state.searchSnapshot = nil  // ✅ 显式清理
    return .merge(
        .cancel(id: CancellationId.typingDebounce),
        .cancel(id: CancellationId.previewSearch),
        .cancel(id: CancellationId.fullSearch),
        .cancel(id: CancellationId.newResultBadgeTimer)
    )

case .fullSearchCancelled:
    state.searchPhase = .cancelled
    state.searchSnapshot = nil  // ✅ 显式清理
    return .none

case .previewSearchFailed(let error):
    state.searchPhase = .failed(message: error.localizedDescription)
    state.searchSnapshot = nil  // ✅ 显式清理
    return .none

case .fullSearchFailed(let error):
    state.searchPhase = .failed(message: error.localizedDescription)
    state.searchSnapshot = nil  // ✅ 显式清理
    return .none

// ✅ searchResultsExceeded 已通过 clearSearchResults() 清理，保持不变
case .searchResultsExceeded(let count, _):
    state.searchPhase = .tooManyResults(currentCount: count, limit: state.searchResultsLimit)
    state.clearSearchResults()  // 包含 searchSnapshot = nil
    return .cancel(id: CancellationId.fullSearch)
```

**优势：**
- ✅ 防止快照泄漏和污染
- ✅ 状态转换更明确
- ✅ 内存管理更可靠

#### P1-3: Full Search 批量查询优化

**问题描述：**

当前设计逐个 session 调用 `searchEvents`（5.4 节 Line 502），如果有 100 个 session，会执行 100 次数据库查询，无法利用数据库查询优化器的批量优化能力。

**当前代码：**

```swift
// ✅ 一次性查询所有 sessions
private func handleFullSearch(
    state: State,
    snapshot: SearchSnapshot,
    environment: Environment
) -> Effect<Action> {
    let resultsLimit = state.searchResultsLimit
    let alreadyFoundCount = state.previewResults.count

    return .run { [dataLoader = environment.dataLoader] send in
        do {
            let totalSessions = snapshot.fullSearchSessions.count

            // ✅ 批量查询所有 fullSearchSessions
            let allMatches = try await dataLoader.searchEvents(
                sessionIds: Set(snapshot.fullSearchSessions.map { $0.id }),
                searchText: snapshot.searchText,
                searchFields: snapshot.searchFields,
                limit: resultsLimit
            )

            // 检查结果总数
            if alreadyFoundCount + allMatches.count > resultsLimit {
                await send(.searchResultsExceeded(
                    currentCount: alreadyFoundCount + allMatches.count,
                    stage: .fullSearch
                ))
                return
            }

            // ✅ 在应用层按 session 分组
            let groupedBySession = Dictionary(grouping: allMatches) { $0.sessionId }

            // ✅ 逐批发送进度更新（模拟渐进式效果）
            for (index, session) in snapshot.fullSearchSessions.enumerated() {
                try Task.checkCancellation()

                let sessionMatches = groupedBySession[session.id] ?? []

                await send(.fullSearchBatchCompleted(
                    newMatches: sessionMatches,
                    currentSessionIndex: index,
                    totalSessions: totalSessions
                ))

                // ✅ 模拟渐进式 UI 更新（50ms 间隔）
                if index < totalSessions - 1 {
                    try await Task.sleep(nanoseconds: 50_000_000)
                }
            }

            await send(.fullSearchCompleted(searchedSessions: totalSessions))

        } catch is CancellationError {
            await send(.fullSearchCancelled)
        } catch {
            await send(.fullSearchFailed(error))
        }
    }
    .cancellable(id: CancellationId.fullSearch)
}
```

```swift
/// 合并两个有序数组（按时间倒序）
private func mergeOrderedResults(
    _ preview: [LogEvent],
    _ full: [LogEvent]
) -> [LogEvent] {
    var result: [LogEvent] = []
    var i = 0, j = 0

    while i < preview.count && j < full.count {
        if preview[i].timestamp > full[j].timestamp {
            result.append(preview[i])
            i += 1
        } else {
            result.append(full[j])
            j += 1
        }
    }

    result.append(contentsOf: preview[i...])
    result.append(contentsOf: full[j...])

    return result
}
```

```swift
private func createSearchSnapshot(...) async throws -> SearchSnapshot {
    // 确定搜索范围
    let searchSessionIds: Set<String>
    if selectedSessionIds.isEmpty || selectedSessionIds.count == allAvailableSessionIds.count {
        searchSessionIds = allAvailableSessionIds
    } else {
        searchSessionIds = selectedSessionIds
    }

    // 获取所有 sessions（按时间倒序）
    let allSessions = try await dataLoader.getSessions(
        sessionIds: searchSessionIds,
        sortBy: .timeDescending
    )

    // ✅ 动态决定 Preview 数量
    var previewSessions: [Session] = []
    var accumulatedEvents = 0

    for session in allSessions {
        previewSessions.append(session)
        accumulatedEvents += session.eventCount

        // 达到以下条件之一则停止：
        // 1. 累积日志数 >= 目标数量
        // 2. session 数量 >= 最大值
        if accumulatedEvents >= SearchPreviewConfig.targetEventCount
            || previewSessions.count >= SearchPreviewConfig.maxSessions {
            break
        }

        // 至少预览最少数量的 session
        if previewSessions.count >= SearchPreviewConfig.minSessions
            && accumulatedEvents >= SearchPreviewConfig.targetEventCount / 2 {
            break
        }
    }

    // 确保至少有 minSessions（如果总数足够）
    if previewSessions.count < SearchPreviewConfig.minSessions {
        previewSessions = Array(allSessions.prefix(SearchPreviewConfig.minSessions))
    }

    let fullSearchSessions = Array(allSessions.dropFirst(previewSessions.count))

    return SearchSnapshot(
        searchText: searchText,
        searchFields: searchFields,
        selectedSessionIds: selectedSessionIds,
        allAvailableSessionIds: allAvailableSessionIds,
        allSessions: allSessions,
        previewSessions: previewSessions,
        fullSearchSessions: fullSearchSessions,
        createdAt: Date()
    )
}
```

```swift
extension DataLoader {
    func searchEvents(
        sessionIds: Set<String>,
        searchText: String,
        searchFields: Set<SearchField>,
        limit: Int
    ) async throws -> [LogEvent] {

        // 构建 WHERE 条件
        var whereClauses: [String] = []
        var arguments: [Any] = []

        // Session 过滤
        let placeholders = sessionIds.map { _ in "?" }.joined(separator: ",")
        whereClauses.append("session_id IN (\(placeholders))")
        arguments.append(contentsOf: sessionIds.map { $0 as Any })

        // 搜索字段过滤
        var searchClauses: [String] = []
        if searchFields.contains(.message) {
            searchClauses.append("message LIKE ?")
            arguments.append("%\(searchText)%" as Any)
        }
        if searchFields.contains(.context) {
            searchClauses.append("context LIKE ?")
            arguments.append("%\(searchText)%" as Any)
        }
        if searchFields.contains(.fileName) {
            searchClauses.append("fileName LIKE ?")
            arguments.append("%\(searchText)%" as Any)
        }
        if searchFields.contains(.function) {
            searchClauses.append("function LIKE ?")
            arguments.append("%\(searchText)%" as Any)
        }

        if !searchClauses.isEmpty {
            whereClauses.append("(\(searchClauses.joined(separator: " OR ")))")
        }

        // 构建完整 SQL
        let sql = """
            SELECT * FROM log_events
            WHERE \(whereClauses.joined(separator: " AND "))
            ORDER BY timestamp DESC
            LIMIT ?
            """
        arguments.append(limit as Any)

        // 执行查询
        return try await database.query(sql, arguments: arguments)
    }
}
```

### 风险控制

**风险 1: 功能丢失**
- **风险**：用户失去快速搜索建议功能
- **缓解**：新功能同样支持快速反馈（Preview 300ms），结果列表中可直接添加筛选器
- **验证**：手动测试常见用户操作流程

**风险 2: 性能下降**
- **风险**：数据库查询慢于内存搜索
- **缓解**：
  - 必须创建索引（`idx_session_timestamp`）
  - 提前测试数据库查询性能
  - P1-3 批量查询优化
- **验证**：性能测试，对比优化前后

**风险 3: UI 破坏**
- **风险**：UI 集成失败，界面错乱
- **缓解**：
  - 逐步替换，先完成 State/Action，再修改 UI
  - 保留现有 UI 布局和交互模式
- **验证**：UI 测试，检查各个状态显示

**风险 4: 数据不一致**
- **风险**：搜索结果与实际日志不一致
- **缓解**：
  - SearchSnapshot 机制保证搜索过程一致性
  - Filter 变化时取消当前搜索
- **验证**：边界情况测试

### 回退方案

如果替换后发现严重问题：

1. **代码回退**
   ```bash
   git checkout feature/deep-search-replace
   git revert <commit-hash>
   # 或
   mv SearchFeature.swift.backup SearchFeature.swift
   mv SearchTypes.swift.backup SearchTypes.swift
   ```

2. **数据库回退**
   - 索引可以保留（不影响现有功能）
   - 如果需要删除：`DROP INDEX idx_session_timestamp`

3. **验证回退成功**
   - 编译通过
   - 搜索建议功能正常
   - 无性能问题
